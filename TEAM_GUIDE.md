## 🧱 1. 团队开发目标
本项目由高级软件工程课程小组共同完成，旨在在现有深度学习教学平台基础上进行功能改进和优化。  
为了保证多人协作的高效与稳定，请所有成员遵守以下 GitHub 工作流程与分支管理规范。

---

## 👥 2. 团队分工与职责
| 角色 | 职责 |
|------|------|
| 组长（牛珂） / 项目管理员 | 负责仓库管理、合并分支、发布版本、任务分配 |
| 前端开发（刘倩文、王冉） | 负责界面优化、交互逻辑、组件重构 |
| 后端开发（牛珂、方妍、王冉） | 负责接口开发、数据库结构调整、服务性能优化 |
| 测试与文档（朱婧怡） | 负责测试用例、Bug 跟踪、撰写文档与报告 |

> 组长应确保每位成员的任务分支独立，定期 review 并合并。

---

## 🌳 3. 分支管理规范

仓库采用 **主干 + 开发 + 功能分支** 模型：

| 分支名称 | 说明 |
|-----------|------|
| `main` | 主分支（仅存放稳定、可发布版本） |
| `dev` | 开发主分支，所有新功能在此合并与测试 |
| `feature-xxx` | 功能分支，每位成员从 `dev` 新建自己的功能分支 |
| `fix-xxx` | Bug 修复分支 |
| `docs` | 文档编写专用分支 |
